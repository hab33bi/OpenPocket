# Water — liquid sim design: **pairwise-hash** particles

Approach: **~320 particles + uniform spatial hash + explicit short-range
pairwise repulsion** (no SPH kernels, no pressure solve). Incompressibility
comes from a cheap linear repulsion impulse applied between neighbours found
in O(1) via the hash, plus a near-density viscosity term. Gravity from the
QMI8658, round-wall projection boundary, jerk→slosh spray, rest breathing
ripple. Everything integer/fixed-point; state lives in `apps::State` (SRAM).

This document is written against the **real** code:
- draw helpers `set_px / blend_px / max_px / soft_dot / fill_rect_black /
  isqrt / pseudo_angle / pseudo_dir / draw_ring_arc` at the bottom of
  `src/scenes/apps.rs`.
- `qmi8658::read_accel(&mut i2c) -> Result<(i16,i16,i16),()>`, 8192 LSB/g @ ±4g.
- `WatchFb::buf_mut() -> &mut [u8]` (RGB565-BE, 466×466×2) + `mark_rect`.
- `lock::NOISE_A / NOISE_B` (`[u8;256]`, already generated by build.rs).
- `wheel::draw_status`, the `azure_lut()`/`saber_lut` color-LUT idiom, and
  the build.rs LUT-emit idiom in `generate_wheel_assets`.
- run loop owns `self.i2c`, `now`, `anim_start`, `wheel_batt`; tick call site
  `apps::tick(&mut self.wfb, idx, &now, elapsed, &mut app_state)` (~L324);
  `frame_us` match (~L457); `flush_dirty` (~L1001); `FRAME_US=50 000`,
  `ANIM_FRAME_US=25 000`.
- `BEZEL_R = 223`, `CX = CY = 233`, `W = H = 466`, `WATER = 7`.

## 1. Fixed-point scheme + overflow proof

### 1.1 Formats and units

| Symbol | Format | Units | Range used |
|---|---|---|---|
| `px[i], py[i]` | `i32` **Q8** | px | 0 … 466 px ⇒ 0 … 119 296 |
| `vx[i], vy[i]` | `i16` **Q8** | px / frame | clamped `±VMAX_Q8 = 18<<8 = 4608` |
| gravity `gx, gy` | `i32` **Q8** | px / frame² | clamped `±GMAX = 4<<8` |
| accel raw | `i16` | LSB (8192 = 1 g) | ±32767 (±4 g) |
| `d2` (pair sep²) | `i32` | px² | 0 … `H2 = 256` (after reject) |
| `PUSH_LUT[d2]` | `i16` | Q7 impulse scalar | 0 … 8192 (build-time clamp) |
| `WATER_LUT[i]` | `(u8,u8)` | RGB565-BE | full |

Everything is frame-based: `dt` is folded into the units, so `pos += v` is a
pure add (no `v*dt` multiply) and `v += g` is a pure add. `dt` never appears in
the hot loop.

Tunable constants (first-flash knobs, edited not rewritten):

```rust
const GRAV_Q8:  i32 = 256;   // Q8 px/frame² at 1 g   (feel: pool crosses ~200px in ~0.5 s at full tilt)
const VMAX_PX:  i32 = 18;    const VMAX_Q8: i32 = VMAX_PX << 8;   // 4608
const GMAX_Q8:  i32 = 4 << 8;            // gravity clamp (±4 g)
const DAMP:     i32 = 250;   // v *= DAMP/256  ≈ 0.977 / frame
const H_PX:     i32 = 16;    const H2: i32 = H_PX * H_PX;         // 256
const PUSH_SHIFT: i32 = 7;   // impulse = (PUSH_LUT[d2]*dxpx) >> 7
const REST_BIAS_CLAMP: i32 = 820;   // ±0.1 g — bias only removes sensor offset, never real tilt
const SLOSH_GAIN: i32 = 220; const JERK_THRESH: i32 = 2600;       // ~0.32 g jerk to spray
```

### 1.2 Overflow proof — every hot-loop multiply

All products are shown at worst-case operand magnitude vs `i32::MAX = 2 147 483 647`.
No `i64` is needed anywhere in the hot loop.

1. **Gravity map** `g = araw * GRAV_Q8 / 8192`
   `|araw| ≤ 32767`, `GRAV_Q8 = 256` → `32767·256 = 8 388 352` ✓ (0.4 % of max).

2. **Jerk→slosh** `s = jerk * SLOSH_GAIN / 8192`
   `|jerk| = |a_now − a_last| ≤ 65534`, `SLOSH_GAIN = 220` → `65534·220 = 14 417 480` ✓.

3. **Integrate** `px += vx`, `vx += gx` — adds only, operands ≤ `VMAX_Q8+GMAX_Q8 ≈ 5632`. No multiply.

4. **Damping** `v * DAMP >> 8`
   `|v| ≤ VMAX_Q8 = 4608`, `DAMP = 250` → `4608·250 = 1 152 000` ✓.

5. **Pair separation²** `d2 = dx*dx + dy*dy`, dx,dy in **px** (`= (px_i−px_j)>>8`)
   Before the `d2 ≥ H2` reject, dx,dy span the 3×3 cell window ⇒ `|dx| ≤ 48`.
   `48·48 = 2304`, `d2 ≤ 4608` ✓ (tiny). After reject, `d2 ∈ 0..255` → LUT index.

6. **Repulsion impulse** `imp = (PUSH_LUT[d2] * dxpx) >> PUSH_SHIFT`
   After reject `|dxpx| < H_PX = 16`; `PUSH_LUT ≤ 8192` → `8192·16 = 131 072` ✓
   `>>7 = 1024` per pair; summed over ≤ ~20 neighbours ≤ `20 480` (accumulated in
   `i32`, then `v` re-clamped to `±VMAX_Q8`). ✓

7. **Viscosity** `visc = ((vj − vi) * VISC_K) >> 8`, folded into the same scan
   `|vj−vi| ≤ 2·VMAX_Q8 = 9216`, `VISC_K ≤ 64` → `9216·64 = 589 824` ✓.

8. **Wall dist²** `r2 = dx*dx + dy*dy`, dx,dy = pos−center in px, `|dx| ≤ 240`
   `240·240 = 57 600`, `r2 ≤ 115 200` ✓.

9. **Wall projection** `((dx * WALL_R) << 8) / r`
   `|dx|·WALL_R ≤ 240·222 = 53 280`; `<<8 = 13 639 680` ✓ (0.6 % of max); `/r`, `r≥222`.

10. **Wall reflection** `vn = (vx*dx + vy*dy) / r`
    `|vx|·|dx| ≤ 4608·240 = 1 105 920`; two terms `≤ 2 211 840`; `/r` ✓.
    `k*dx` where `k = (vn*(256+e))>>8`, `|vn| ≤ 2 211 840/222 ≈ 9963`,
    `k ≤ (9963·282)>>8 ≈ 10 976`, `k·dx ≤ 10 976·240 = 2 634 240` ✓.

11. **Render speed term** `spd2 = vx*vx + vy*vy`
    `4608·4608 = 21 233 664`, `×2 = 42 467 328` ✓ (2 % of max). Then `isqrt`.

12. **Cell index** `(py>>4)*GRID_W + (px>>4)` → `29·30 + 29 = 899 < 900` ✓.

Largest product anywhere is **~42.5 M** (< 2 % of `i32::MAX`); comfortable margin.
`isqrt` is only ever called on values ≤ 115 200 (wall) or ≤ 42.5 M (speed, once
per drawn particle) — never per pair (that's the whole point of `PUSH_LUT`).

## 2. State layout + SRAM byte count

All simulation state is a `Water` sub-struct **inside `apps::State`** (the
run-loop-owned instance) — internal SRAM, random-access, never the PSRAM
framebuffer. `const fn new()` zero-inits every array.

```rust
const NW: usize     = 320;
const GRID_W: i32   = 30;
const NCELL: usize  = (GRID_W * GRID_W) as usize;   // 900

pub struct Water {
    px:  [i32; NW],           // 1280  Q8 position x
    py:  [i32; NW],           // 1280  Q8 position y
    vx:  [i16; NW],           //  640  Q8 velocity x
    vy:  [i16; NW],           //  640  Q8 velocity y
    nbr: [u8;  NW],           //  320  neighbour count (surface/meniscus flag)
    cell_start: [u16; NCELL+1], // 1802  CSR row starts (immutable during relax)
    cursor:     [u16; NCELL+1], // 1802  scatter cursor (copy of cell_start)
    order:      [u16; NW],      //  640  particles sorted by cell
    bias_x: i32, bias_y: i32,   //    8  rest calibration (clamped ±0.1 g)
    last_ax: i16, last_ay: i16, last_az: i16, // 6  previous accel (jerk)
    rng: u32,                   //    4  xorshift for spawn/spray
    phase: u32,                 //    4  breathing clock
    last_bbox: (i16,i16,i16,i16), // 8  previous draw bbox (erase union)
    seeded: bool,               //    1
}
// total ≈ 8 285 bytes ≈ 8.1 KB  (+ struct padding ≈ 8.3 KB)
```

`PUSH_LUT` (`[i16;257]`, 514 B) and `WATER_LUT` (`[(u8,u8);256]`, 512 B) are
**build-time consts in flash/.rodata**, not SRAM state.

`apps::State` gains one field:

```rust
pub struct State {
    // ... existing fields ...
    pub wa: Water,
}
// State::new():   wa: Water::new(),
```

> **Stack note.** `app_state` is a `run()` local (the loop is `-> !`, so it
> lives forever on that stack). +8.3 KB there is fine on the ESP32-S3 main
> stack, but confirm headroom on first flash; if tight, promote `Water` to a
> `static` guarded by the single-core run loop. The brief permits stack *or*
> static — field-in-`State` is the default per the doctrine.

## 3. Per-frame update

### 3.1 Gravity from the IMU (axis map + calibration)

Read happens in the **run loop** (owns `self.i2c`), raw vector handed to the
tick. Axis→screen mapping is the one true unknown (like the touch Y-flip) and
is exposed as first-flash const edits:

```rust
// FIRST-FLASH TUNABLES — measured on-device by logging (ax,ay,az) while
// tilting each screen edge down, exactly like the touch driver's flip.
const IMU_X_SRC: usize = 0;  const IMU_X_SGN: i32 = 1;  // which raw axis → screen +x (right)
const IMU_Y_SRC: usize = 1;  const IMU_Y_SGN: i32 = 1;  // which raw axis → screen +y (down)

#[inline] fn axis(a:(i16,i16,i16), s:usize)->i32 { match s {0=>a.0 as i32,1=>a.1 as i32,_=>a.2 as i32} }

fn map_gravity(a:(i16,i16,i16), bias_x:i32, bias_y:i32) -> (i32,i32) {
    let gx_raw = IMU_X_SGN*axis(a,IMU_X_SRC) - bias_x;
    let gy_raw = IMU_Y_SGN*axis(a,IMU_Y_SRC) - bias_y;
    ( ((gx_raw*GRAV_Q8)/8192).clamp(-GMAX_Q8,GMAX_Q8),
      ((gy_raw*GRAV_Q8)/8192).clamp(-GMAX_Q8,GMAX_Q8) )
}
```

**Rest calibration** (captured on app open, in the first tick that has a live
IMU sample): average the in-plane reading and store it as `bias`, **clamped to
±0.1 g** (`REST_BIAS_CLAMP=820`). The clamp is deliberate: it removes only the
QMI8658's small zero-g offset, and can *never* cancel a real tilt. Consequence
— a **flat, face-up** watch reads in-plane g ≈ 0 (physically correct: gravity
is along z, into the screen), so the pool sits and *breathes*; **tilting** is
what makes it run downhill and climb the far wall. This matches the vibe doc
("hold it vertical and it all pours to one side"). `az` is unused by the 2-D
sim except to detect flat-vs-upright for modulating breathing/splash.

On IMU read error the sim reuses `last_a*` (or a fixed down-vector) — the app
never stalls on a dead sensor.

### 3.2 Physics core — spatial hash + pairwise repulsion

Order per frame: **integrate → build hash → relax (repulsion+viscosity+surface)
→ damp → wall.**

**Integrate** (semi-implicit Euler, all Q8):
`v += g (+ breathing + slosh)`, clamp `v`, `pos += v`.

**Hash build** (count-sort CSR, ~`2N + NCELL` ops): count particles per cell →
prefix sum → scatter indices into `order[]` using a `cursor` copy so
`cell_start` stays intact for the relax scan.

**Relax** — the incompressibility. For each particle `i`, scan the 3×3 block of
cells around it (`cell = H_PX`, so *every* neighbour within `H_PX` is inside the
block by construction). For each neighbour `j≠i` with `d2 < H2`:
- **repulsion** impulse `PUSH_LUT[d2]·(dx,dy)>>7` pushes `i` away from `j`;
- **viscosity** nudges `v_i` toward `v_j` (near-density smoothing);
- **`nbr[i]`** counts in-radius neighbours (free surface detector for meniscus).

`PUSH_LUT[d2] = clamp( STIFF·(H−d)/d , 0, 8192 )` (build-time, `d=√d2`) — bakes
the `1/d` normalization **and** the linear falloff into one table lookup, so the
hot pair loop has **no sqrt and no divide**: one load + two multiplies + two
shifts. Near `d→0` the entry is clamped, which is the anti-singularity guarantee
(overlapping particles get a strong *bounded* push). We apply half the impulse
to `i` (each pair is visited from both ends → symmetric, no `j>i` bookkeeping).
**One relaxation pass** per frame (two when calm and cheap) — honestly grainier
than a pressure projection, but the neon-square aesthetic hides it and viscosity
smooths the surface.

**Damping** `v *= DAMP/256` — global energy sink so motion always settles.

**Round wall** (no leaks): if `r² > WALL_R²`, `isqrt` once, **project position
exactly onto `r = WALL_R`**, and remove the outward normal velocity component
with restitution `e≈0.1` (energy loss, no perpetual bounce). This is a hard
positional constraint, not a soft collider — nothing can be outside the circle
at end of frame. `VMAX_PX=18 < WALL_R` interior margin, so a particle can't jump
the wall faster than the per-frame check catches it.

### 3.3 Jerk → slosh / spray

Frame-to-frame accel delta on the two in-plane axes is the jerk. When
`|jerk_x|+|jerk_y| > JERK_THRESH` (a flick), inject `jerk·SLOSH_GAIN/8192` into
**all** particles (whole-body slosh) and add an extra randomized perpendicular
kick to **surface** particles (`nbr[i] < SURF_NBR`) — that is the wave breaking
and throwing spray. Bounded by the same `v` clamp, so a hard shake can't blow up.

### 3.4 Rest breathing ripple

A slow triangle (`phase`, same tempo idiom as `breath()` in apps.rs) adds a tiny
oscillating vertical acceleration (`±~0.1 px/frame²`), **scaled down as in-plane
`|g|` grows** so it's a rest-only shimmer — the liquid is alive even flat, but
tilt dominates when tilted. One LUT/triangle read, applied as a shared constant
to all particles (negligible cost).

## 4. Rendering

### 4.1 The look

Each particle is a **4×4 neon-blue filled square** whose color is
`WATER_LUT[idx]` (deep indigo → blue → neon-cyan → white). The index is:

```
idx = base(surface?210:150) + speed_term + aurora − depth_term   (clamped 0..255)
  speed_term = isqrt(vx²+vy²) >> 5     // fast/crest → brighter
  depth_term = nbr[i] * 3              // deeper (more neighbours) → darker indigo body
  aurora     = (NOISE_A[(x>>2)+t1] + NOISE_B[(y>>2)−t2] − 128) >> 3   // ±~16 drift
```

- **WATER_LUT shading by speed/depth**: crests and spray glow white, the settled
  body is deep indigo — one gradient, the saber-LUT doctrine.
- **Aurora drift**: reuses `lock::NOISE_A/NOISE_B` counter-scrolled and sampled
  by *position*, so a luminous flow moves through the mass (ties Water into the
  OS visual language; zero per-frame generation cost).
- **Dithering (lit pixels only)**: a 4×4 Bayer offset is added to the *LUT
  index* before lookup, killing RGB565 banding on the gradient without touching
  black background — exactly the build.rs `GLOW_RING` doctrine.
- **Meniscus**: surface particles (`nbr[i] < SURF_NBR`, free from the relax
  scan) render brighter (`base=210`) **and** get a soft `soft_dot(r=3)` neon-
  white glow — a light-catching waterline that sparkles where the surface is
  exposed or spraying. Body particles get the flat square only (cheap).
- Body squares **SET** the LUT pair (overlaps overwrite → the dense pool reads
  as a solid dithered cyan sheet, i.e. *liquid*, not dots); only crest glows use
  `max_px` (merge, never self-stack). Because we clear+redraw fresh each frame,
  SET is idempotent.

### 4.2 Damage / flush story (and the clock)

The framebuffer is **retained**, so we do **not** memset 424 KB/frame. Each
frame:
1. compute the union of *this* frame's particle bbox and `last_bbox`;
2. `fill_rect_black` that union (clamped to `y ≥ CLOCK_BOT=70`);
3. redraw all particles into it;
4. `mark_rect(union)`; store this frame's bbox as `last_bbox`.

`flush_dirty` then does the right thing automatically: when the pool is compact
(union < ¾ frame) it **partial-flushes** just the union (cheaper than 13 ms,
higher fps); only when the liquid sheets across most of the face does it cross
the ¾ threshold and **full-flush (13 ms)**. So the app is *adaptive*: partial
when pooled, **full-frame in the worst case** — the ~40 fps ceiling is the
worst-case guarantee, better when calm. This reconciles with the brief's
"full-frame-flush app" framing (worst case) while spending less most frames.

**Status clock stays intact**: all liquid writes are clipped to `y ≥ CLOCK_BOT`
(the clock band `y 26..66` plus margin) and the clear rect's `y0` is clamped the
same way. The retained clock is never overwritten; on a high tilt the water
simply rises *behind* the clock (a hard top line at y=70). Minute rollover is
handled by the existing run-loop `tick_status` path (`shows_status(WATER)=true`).

### 5.5 `build.rs` — `WATER_LUT` + `PUSH_LUT` generator

Register `generate_water_lut()` in `main()` and `include!` it in apps.rs:
`include!(concat!(env!("OUT_DIR"), "/water_lut.rs"));`

```rust
fn generate_water_lut() {
    // ---- color LUT: deep indigo → blue → neon-cyan → white (RGB565-BE pairs) ----
    let stops: [(i32,(i32,i32,i32));5] = [
        (0,   (2,  4,  22)),   // deep body (near-black indigo)
        (72,  (0,  46, 130)),  // deep blue
        (150, (0,  150,235)),  // signature neon blue
        (208, (120,226,255)),  // bright cyan crest
        (255, (232,250,255)),  // white spray
    ];
    let lerp=|a:i32,b:i32,t:i32,d:i32| a+(b-a)*t/d;
    let mut body = String::from("/// Auto-generated by build.rs. Do not edit.\n");
    body.push_str("pub static WATER_LUT: [(u8,u8);256] = [");
    for i in 0..256i32 {
        let mut s=0; while s+1<stops.len() && i>stops[s+1].0 { s+=1; }
        let (i0,c0)=stops[s]; let (i1,c1)=stops[(s+1).min(4)];
        let d=(i1-i0).max(1); let t=i-i0;
        let r=lerp(c0.0,c1.0,t,d); let g=lerp(c0.1,c1.1,t,d); let b=lerp(c0.2,c1.2,t,d);
        let r5=(r.clamp(0,255) as u16*31/255)&0x1F;
        let g6=(g.clamp(0,255) as u16*63/255)&0x3F;
        let b5=(b.clamp(0,255) as u16*31/255)&0x1F;
        let px=(r5<<11)|(g6<<5)|b5;
        if i>0 { body.push(','); }
        body.push_str(&format!("({},{})",(px>>8) as u8, px as u8));
    }
    body.push_str("];\n");

    // ---- repulsion LUT: index d2 (0..=256) → clamp(STIFF*(H-d)/d, 0, 8192) ----
    const H:f64=16.0; const STIFF:f64=1600.0;
    body.push_str("pub static PUSH_LUT: [i16;257] = [");
    for d2 in 0..=256usize {
        let d=(d2 as f64).sqrt();
        let v = if d<1.0 { STIFF*(H-1.0) }            // near-coincident: strong but clamped
                else if d>=H { 0.0 }
                else { STIFF*(H-d)/d };
        if d2>0 { body.push(','); }
        body.push_str(&format!("{}", (v as i32).clamp(0,8192)));
    }
    body.push_str("];\n");

    let out=std::env::var("OUT_DIR").unwrap();
    std::fs::write(std::path::Path::new(&out).join("water_lut.rs"), body).unwrap();
}
```

## 6. Per-frame ops + ms (physics vs render), particle fit

At 240 MHz, all state in SRAM (no cache miss), ~4–6 cyc / int op amortized.

**Physics (N=320, ~19 candidates/particle):**

| Stage | ops |
|---|---|
| integrate + gravity + breathing | 320 × 18 ≈ 5 760 |
| hash build (count+prefix+scatter) | 2·320 + 900 ≈ 1 540 |
| relax (repulsion+viscosity+nbr) | 320 × 19 × 14 ≈ **85 100** |
| damping | 320 × 4 ≈ 1 280 |
| wall (all r², ~30 project) | 320×6 + 30×40 ≈ 3 120 |
| **total** | **≈ 96 800 ops → ~2.5 ms** |

The relax pass dominates and is *sqrt-free* (that's the `PUSH_LUT` payoff);
`isqrt` runs only in the wall (~30×) and once per drawn particle for the speed
term.

**Render (clear-bound):**

| Stage | cost |
|---|---|
| clear union bbox (pooled ~420×140 = 117 KB) | ~2–3 ms (worst tilt ~7 ms) |
| draw 320 squares (16 px SET+dither) | ~0.6 ms |
| ~60 surface glows (`soft_dot r3`) | ~0.3 ms |
| **total** | **~3 ms typ / ~7 ms worst** |

**Frame:** typical `2.5 + 3 + partial-flush(~6) ≈ 12 ms`; worst
`2.5 + 7 + full-flush(13) ≈ 22.5 ms` — both inside the **25 ms** cadence, so
**40 fps holds even worst-case**, capped by the flush floor exactly as the
perf plan predicts.

**Particle fit for ~10 ms compute budget:** physics scales ~linearly in N
(render clear is bbox-bound, not N-bound), so N could rise to **~450** and still
fit ~4 ms physics. **Call: N = 320** as the premium-stable default (spend the
margin on physics quality/stability, not raw count, per the plan) with a clean
path to 400 if the surface wants more density. With perf-plan **P1 async flush**
later, the flush leaves the critical path and this same sim could push toward
55–70 fps.

---

## 7. Stability guarantees + honest weaknesses

### Guarantees
- **Never blows up.** `v` is hard-clamped to `±VMAX_Q8` every frame (CFL: ≤ ~1
  cell/frame); `DAMP<1` bleeds energy; `PUSH_LUT` is build-time clamped to 8192
  *and* clamped for `d→0`, so no unbounded force exists. Energy is bounded ⇒ it
  always settles.
- **Never leaks.** The wall is a **positional projection** to exactly `r=WALL_R`
  applied to every out-of-bounds particle each frame (not a thin collider),
  with `e<1` restitution on the outward normal. `VMAX_PX < WALL_R` interior
  margin ⇒ no tunneling. Nothing is ever outside the circle at frame end.
- **Never piles to a singularity.** Pairwise repulsion enforces a minimum
  spacing; the `d→0` clamp guarantees even a direct overlap resolves in a few
  frames without exploding. Unlike gravity-only "balls," the mass *spreads and
  levels* — it behaves incompressibly at the UI level.
- **Hash always valid.** Rebuilt every frame *before* relax, `cell = H_PX` ⇒
  every in-radius neighbour is inside the 3×3 block by construction; the scan is
  never wrong (only positions from this frame are used).

### Honest weaknesses (specific to pairwise-hash, no pressure solve)
1. **Grainy / "boiling" surface.** One repulsion pass + no pressure projection
   gives a jitterier surface than PIC/FLIP or double-density relaxation. The
   viscosity term and the stylized neon squares mask it, but a perfectly glassy
   calm surface is harder than a height-field would give.
2. **Slight compressibility.** With 1 pass, the pool compresses a little under
   strong gravity (bottom denser than rest spacing, surface dips a hair). More
   passes fix it at CPU cost; we accept 1 (2 when calm).
3. **No hard volume conservation.** Total footprint can "breathe" slightly — not
   a strict incompressible constraint. Visually fine.
4. **Meniscus flicker.** `nbr` threshold surface detection can toggle on
   borderline particles; a small hysteresis on `SURF_NBR` is the mitigation.
5. **Spray is droplets, not sheets.** Thrown particles don't fragment or form
   thin films — reads as droplets, acceptable for the style.
6. **Coarse at low density.** 320 particles is a chunky fluid; a single drop or
   thin film looks blocky — the 4 px squares hide it, but it's a real ceiling.
7. **Gauss-Seidel order bias.** Applying impulses in array order is mildly
   asymmetric; negligible visually, and the count-sort reshuffles order spatially
   each frame anyway.

## 8. Core Rust sketch (real bodies, repo helpers)

```rust
// ---- constants (see §1.1) + CLOCK_BOT / SURF_NBR ----
const WALL_R: i32 = 222; const WALL_R2: i32 = WALL_R*WALL_R;
const CLOCK_BOT: i32 = 70; const SURF_NBR: u8 = 4; const VISC_K: i32 = 40;
const BAYER4: [[i32;4];4] = [[-8,0,-6,2],[4,-4,6,-2],[-5,3,-7,1],[7,-1,5,-3]];

include!(concat!(env!("OUT_DIR"), "/water_lut.rs"));   // WATER_LUT, PUSH_LUT

#[inline] fn rng(s:&mut u32)->u32{ let mut x=*s; x^=x<<13; x^=x>>17; x^=x<<5; *s=x; x }
#[inline] fn axis(a:(i16,i16,i16), s:usize)->i32{ match s {0=>a.0 as i32,1=>a.1 as i32,_=>a.2 as i32} }
#[inline] fn cell_of(px:i32,py:i32)->usize{
    let x=(px>>8).clamp(0,W-1); let y=(py>>8).clamp(0,H-1);
    ((y>>4)*GRID_W + (x>>4)) as usize
}
#[inline] fn put565(fb:&mut [u8], x:i32,y:i32, hi:u8,lo:u8){
    if x<0||x>=W||y<CLOCK_BOT||y>=H { return; }
    let i=((y*W+x)*2) as usize; if i+1<fb.len(){ fb[i]=hi; fb[i+1]=lo; }
}

// ---------- entry (called from the run loop, §5.4) ----------
pub fn water_tick(wfb:&mut WatchFb, imu:Option<(i16,i16,i16)>, _now:&WallTime, elapsed_ms:u32, st:&mut State){
    let w = &mut st.wa;
    let acc = imu.unwrap_or((w.last_ax, w.last_ay, w.last_az));

    if !w.seeded {                                   // first tick after reveal: calibrate + spawn
        w.bias_x = (IMU_X_SGN*axis(acc,IMU_X_SRC)).clamp(-REST_BIAS_CLAMP, REST_BIAS_CLAMP);
        w.bias_y = (IMU_Y_SGN*axis(acc,IMU_Y_SRC)).clamp(-REST_BIAS_CLAMP, REST_BIAS_CLAMP);
        w.rng = 0x2545_F491 ^ elapsed_ms.wrapping_mul(2_654_435_761);
        for i in 0..NW {
            let rx = (rng(&mut w.rng)%300) as i32 - 150;
            let ry = (rng(&mut w.rng)%120) as i32 - 190;   // top region
            w.px[i]=(CX+rx)<<8; w.py[i]=(CY+ry)<<8; w.vx[i]=0; w.vy[i]=256;
        }
        w.last_ax=acc.0; w.last_ay=acc.1; w.last_az=acc.2;
        w.seeded=true; w.last_bbox=(0,0,-1,-1);
    }

    // jerk -> slosh (whole-body impulse this frame)
    let jx = IMU_X_SGN*axis(acc,IMU_X_SRC) - IMU_X_SGN*axis((w.last_ax,w.last_ay,w.last_az),IMU_X_SRC);
    let jy = IMU_Y_SGN*axis(acc,IMU_Y_SRC) - IMU_Y_SGN*axis((w.last_ax,w.last_ay,w.last_az),IMU_Y_SRC);
    let slosh = if jx.abs()+jy.abs() > JERK_THRESH {
        ((jx*SLOSH_GAIN/8192).clamp(-VMAX_Q8,VMAX_Q8), (jy*SLOSH_GAIN/8192).clamp(-VMAX_Q8,VMAX_Q8))
    } else { (0,0) };
    w.last_ax=acc.0; w.last_ay=acc.1; w.last_az=acc.2;

    let (gx,gy) = map_gravity(acc, w.bias_x, w.bias_y);
    // breathing: triangle, only meaningful when nearly flat (|g| small)
    w.phase = w.phase.wrapping_add(1);
    let tri = { let p=(w.phase/2)%512; if p<256 {p as i32} else {511-p as i32} };  // 0..255
    let breathe = if gx.abs()+gy.abs() < 200 { (tri-128)/6 } else { 0 };

    step(w, gx, gy+breathe, slosh);
    render(wfb, w, elapsed_ms);
}

// ---------- physics ----------
fn step(w:&mut Water, gx:i32, gy:i32, slosh:(i32,i32)){
    for i in 0..NW {                                   // integrate
        let vx=((w.vx[i] as i32)+gx+slosh.0).clamp(-VMAX_Q8,VMAX_Q8);
        let vy=((w.vy[i] as i32)+gy+slosh.1).clamp(-VMAX_Q8,VMAX_Q8);
        w.px[i]+=vx; w.py[i]+=vy; w.vx[i]=vx as i16; w.vy[i]=vy as i16;
    }
    build_hash(w);
    relax(w);
    for i in 0..NW {                                   // damp
        w.vx[i]=(((w.vx[i] as i32)*DAMP)>>8) as i16;
        w.vy[i]=(((w.vy[i] as i32)*DAMP)>>8) as i16;
    }
    for i in 0..NW { wall(w,i); }
}

fn build_hash(w:&mut Water){
    for c in 0..=NCELL { w.cell_start[c]=0; }
    for i in 0..NW { w.cell_start[cell_of(w.px[i],w.py[i])+1]+=1; }   // count
    for c in 0..NCELL { w.cell_start[c+1]+=w.cell_start[c]; }         // prefix sum
    for c in 0..=NCELL { w.cursor[c]=w.cell_start[c]; }               // cursor copy
    for i in 0..NW {                                                  // scatter
        let c=cell_of(w.px[i],w.py[i]);
        let s=w.cursor[c]; w.order[s as usize]=i as u16; w.cursor[c]=s+1;
    }
}

fn relax(w:&mut Water){
    for i in 0..NW {
        let xi=w.px[i]>>8; let yi=w.py[i]>>8;
        let cx=((xi>>4).clamp(0,GRID_W-1)); let cy=((yi>>4).clamp(0,GRID_W-1));
        let (mut ax, mut ay, mut n)=(0i32,0i32,0u8);
        let mut gyc=(cy-1).max(0);
        while gyc<=(cy+1).min(GRID_W-1) {
            let mut gxc=(cx-1).max(0);
            while gxc<=(cx+1).min(GRID_W-1) {
                let c=(gyc*GRID_W+gxc) as usize;
                for s in w.cell_start[c]..w.cell_start[c+1] {
                    let j=w.order[s as usize] as usize; if j==i { continue; }
                    let dx=xi-(w.px[j]>>8); let dy=yi-(w.py[j]>>8);
                    let d2=dx*dx+dy*dy;
                    if d2>=H2 || d2==0 { continue; }
                    n+=1;
                    let push=PUSH_LUT[d2 as usize] as i32;             // sqrt-free, /d baked in
                    ax += (push*dx)>>PUSH_SHIFT;
                    ay += (push*dy)>>PUSH_SHIFT;
                    // near-density viscosity (smooths the grainy surface)
                    ax += (((w.vx[j] as i32)-(w.vx[i] as i32))*VISC_K)>>8;
                    ay += (((w.vy[j] as i32)-(w.vy[i] as i32))*VISC_K)>>8;
                }
                gxc+=1;
            }
            gyc+=1;
        }
        w.nbr[i]=n;
        // half the correction to i (pair seen from both ends) + re-clamp
        w.vx[i]=(((w.vx[i] as i32)+(ax>>1)).clamp(-VMAX_Q8,VMAX_Q8)) as i16;
        w.vy[i]=(((w.vy[i] as i32)+(ay>>1)).clamp(-VMAX_Q8,VMAX_Q8)) as i16;
    }
}

fn wall(w:&mut Water, i:usize){
    let dx=(w.px[i]>>8)-CX; let dy=(w.py[i]>>8)-CY;
    let r2=dx*dx+dy*dy; if r2<=WALL_R2 { return; }
    let r=isqrt(r2 as u32) as i32;                                    // one sqrt, boundary only
    w.px[i]=(CX<<8) + ((dx*WALL_R)<<8)/r;                             // project onto rim (Q8)
    w.py[i]=(CY<<8) + ((dy*WALL_R)<<8)/r;
    let vx=w.vx[i] as i32; let vy=w.vy[i] as i32;
    let vn=(vx*dx+vy*dy)/r;                                           // outward normal speed (Q8)
    if vn>0 {
        let k=(vn*(256+26))>>8;                                       // (1+e), e≈0.1
        w.vx[i]=(vx-(k*dx)/r) as i16;
        w.vy[i]=(vy-(k*dy)/r) as i16;
    }
}

// ---------- render (§4) ----------
fn render(wfb:&mut WatchFb, w:&mut Water, elapsed_ms:u32){
    let (mut b0,mut b1,mut b2,mut b3)=(i32::MAX,i32::MAX,i32::MIN,i32::MIN);
    for i in 0..NW { let x=w.px[i]>>8; let y=w.py[i]>>8;
        if x<b0{b0=x} if x>b2{b2=x} if y<b1{b1=y} if y>b3{b3=y} }
    let cur=(b0-4,(b1-4).max(CLOCK_BOT),b2+4,b3+4);
    let lb=w.last_bbox;
    let er=(cur.0.min(lb.0 as i32), cur.1.min((lb.1 as i32).max(CLOCK_BOT)),
            cur.2.max(lb.2 as i32), cur.3.max(lb.3 as i32));
    let t1=(elapsed_ms>>5) as i32; let t2=(elapsed_ms>>6) as i32;
    {
        let fb=wfb.buf_mut();
        fill_rect_black(fb, er.0, er.1.max(CLOCK_BOT), er.2, er.3);   // erase (keeps clock band)
        for i in 0..NW {
            let x=w.px[i]>>8; let y=w.py[i]>>8;
            let spd=isqrt(((w.vx[i] as i32).pow(2)+(w.vy[i] as i32).pow(2)) as u32) as i32;
            let surf = w.nbr[i] < SURF_NBR;
            let aur=(lock::NOISE_A[(((x>>2)+t1) & 255) as usize] as i32
                    +lock::NOISE_B[(((y>>2)-t2) & 255) as usize] as i32 - 128) >> 3;
            let idx=((if surf {210} else {150}) + (spd>>5) + aur - (w.nbr[i] as i32*3)).clamp(0,255);
            for dy in 0..4 { let yy=y-2+dy;
                for dx in 0..4 { let xx=x-2+dx;
                    let li=(idx + BAYER4[(yy&3) as usize][(xx&3) as usize]).clamp(0,255) as usize;
                    let (hi,lo)=WATER_LUT[li]; put565(fb, xx, yy, hi, lo);
                }
            }
            if surf { soft_dot(fb, x, y, 3, (0,210,255), 150); }      // meniscus/crest spark
        }
    }
    w.last_bbox=(cur.0 as i16, cur.1 as i16, cur.2 as i16, cur.3 as i16);
    wfb.mark_rect(er.0, er.1, er.2, er.3);                            // partial when pooled, full on big slosh
}
```
