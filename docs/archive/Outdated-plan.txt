Initial technical design:
- Use a custom scene renderer.
- Use RGB565 unless a different format is required by the display path.
- Use PSRAM for full framebuffers and simulation buffers where appropriate.
- Keep DMA-capable internal memory for flush chunks if required by the HAL/display peripheral.
- 466 x 466 x 2 bytes is about 434 KB per RGB565 framebuffer; double buffering is about 868 KB, which fits in 8 MB PSRAM but must be tested for speed.

- Prefer dirty-region or partial redraw when it helps.
- For highly animated scenes, full-frame flush may be acceptable if frame pacing is stable.
- Measure before optimising.
